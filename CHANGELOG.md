# Changelog

All notable updates to Auto PTU are tracked here so new contributors can see how the rules engine grows over time. Follow [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) style with PTU rulebook citations whenever mechanics change.

## [Unreleased] ? v0.5 "Statuses, Abilities, and Items"

- Web battle UX and AI orchestration upgraded: added AI model registry endpoints (`/api/ai/models`, `/api/ai/models/select`) with automatic model versioning from learning-drift math, new Rules-safe AI diagnostics panel (legal action count/top choices/selected score/fallback reason), multi-side random/royale start guards that ignore 2-side roster injection, and cinematic auto-camera controls (director profiles, smooth lock/zoom phases, adaptive pacing, replay export, and queue/perf backpressure so auto-step never outruns camera/animations).
- Documentation cleanup: consolidated canonical docs in `DOCS_INDEX.md`, archived duplicates, and clarified PTUDatabase/CSV source-of-truth guidance.
- Item effects added: Black Augurite (Splinter on contact), Blue Orb (primal reversion ready), Boggy Clay (fog immunity), Buckler Shield (Parry), Breastplate/Buff Coat (speed/resistance + mitigation), plus non-combat placeholders (Body, Body + Head, Bounce Case).
- Added phase-1 trainer class/feature scaffolding: new serializable `TrainerClassSpec`/`TrainerFeatureSpec` data schemas, `TrainerSideSpec` metadata plumbing, and a runtime `TrainerFeatureDispatcher` that hooks round/phase/action triggers with frequency, cooldown, and resource gates plus generic effect payload support.
- Expanded trainer class/feature scaffolding to phase 2: `TrainerFeatureSpec` now carries declarative `conditions`, the dispatcher now evaluates context filters (actor scope, action/move filters, once-per-actor-per-round, chance, damage/round windows), supports richer target scopes, and applies multi-effect payload batches (`heal`, `raise_cs`, `apply_status`, `remove_status`, `set_weather`, trainer AP grants) with structured trainer-feature log entries.
- Added phase-3 trainer feature prerequisite gating: feature specs can now declare `min_trainer_level`, required class/subclass, and required feature chains; runtime dispatch enforces those prerequisites before trigger evaluation so class progression trees can be modeled safely in future trainer-content batches.
- AI-vs-AI batch simulations now persist to disk by default: `run_ai_battles` writes `results.jsonl` + `summary.json` under `reports/simulations/ai_batch_*`, records winner/error/turn counts plus move/status/hazard/ability summaries per battle, and supports optional full log storage for deeper offline analysis.
- Expanded launcher AI-vs-AI controls: batch size cap raised to 5000 battles, worker cap set to 16 on Windows (plus runtime clamping by CPU/job count and BLAS thread pinning to prevent paging/memory crashes), and depth/top-k/top-m/rollout parameters are now prompted and passed through to both watch-mode and batch simulations.
- Added a first-class optimizer contract in `auto_ptu/optimizer.py` with JSON-ready `EncounterModel` and `BuildGenome` schemas plus a deterministic proxy scorer that blends offense, defense, tempo, and consistency against weighted enemy archetypes.
- Random battle level-up stat assignment now uses an objective-driven optimizer (offense, bulk, speed/evasion breakpoints, move reliability, and setup pressure) per archetype, then enforces HP floors, so generated builds stay role-consistent while scaling better at high levels.
- Random CSV battles now enforce a level-scaled HP investment floor during level-up stat allocation, so high-level rosters retain meaningful bulk while keeping the same minmax build logic across all levels.
- Auto PTU now runs `auto_ptu/tools/auto_update.py` at launcher/CLI startup so `ABILITY_LOG.md` auto-regenerates whenever the Fancy PTU CSV or the generator script change, keeping the master ability checklist current without extra commands.
- The grid renderer tolerates dict-based tile metadata (not just string tile types) and `BattleState._normalize_hazard_layers` lives directly on `BattleState`, preventing the previously logged `'dict' object has no attribute 'lower'` crash during the `levitate_demo` hazard walkthrough.
- Web UI hover cards are now fixed-position, scrollable tooltips with full move rules text plus ability/item/nature descriptions on hover; this removes layout jumps when browsing long move text and keeps battle controls stable while reading details.
- AI battle pacing now considers setup/status value before auto-committing to damage every turn; opening rounds can choose buffs/debuffs when they score better than weak damage and still preserve Struggle as last resort.
- Move hover popup behavior was stabilized across live rerenders (poll refresh), using a compact text-first tooltip that no longer jitters or loses scroll focus while reading long descriptions.
- Status log lines now always surface the specific status name when available, and battle feed keywords (statuses/moves/abilities/items) now expose hover tooltips so players can inspect what each keyword means directly from the log.
- Move hover now works even when actions are not currently usable (non-actionable move buttons stay hoverable instead of HTML-disabled), and the popup was simplified to a pure text description panel like ability/item tooltips.
- Ability tooltip metadata now prioritizes PTU CSV ability text over PokeAPI text, including fallback for non-PokeAPI abilities; cached placeholder entries (effect text matching the ability name) are automatically repaired.
- Web tooltip rendering now treats placeholder effect strings (`"-"`, `"n/a"`, or exact name echoes such as `"Haunting"`) as unavailable so the UI no longer shows self-name pseudo-descriptions.
- Move hover now uses the same popup tooltip path as ability/item hover (anchored floating overlay), and the tooltip node is attached at `document.body` to prevent inline "square" layout shifts inside the move panel.
- Move hover text is now a full mechanics breakdown (type/category, AC gate, DB handling, range/target/frequency/priority, keywords, PTU rule text, and optional reference text), and move log lines now include richer combat bits (AC roll/needed, DB, damage roll/base, multipliers, Atk vs Def terms, crit extras, and multi-hit counts when available).

- Persistent statuses (Burn, Poison, Paralysis, Sleep, Freeze, Badly Poisoned) now track timers, skips, thaw/damage wakes, stacking ticks, forced pushes, and friendly-fire warnings; the `status_demo`, `sleep_demo`, `freeze_demo`, and `poison_demo` launcher entries let QA replay each scenario on demand.
- Phase-aware Burn/Poison ticks and the Poison Heal/Shed Skin overrides log every conversion so PTU Core 1.05 (Chapter 5) damage flows stay transparent.
- Ability labs now support `ability_demo` (Poison Heal), plus the new `shed_skin_demo`, `swift_swim_demo`, and `chlorophyll_demo`; the README/launcher call them out and `ABILITY_HOOKS.md` shows how far along the first 10 CSV abilities are.
- Ground hazards (Spikes, Toxic Spikes, Stealth Rock, Sticky Web) now sit on tiles, log damage/status/speed events at the start of every grounded Pokemon's turn, and Levitate/`levitate_demo` demonstrate the immunity to those hazards and to ground-type moves.
- Target selection prints a color-coded "in range / LOS blocked / out of range" table plus explicit ally warnings, and the `targeting_demo` campaign + launcher option exercises the new cues alongside regression tests.
- Launcher menu, README, and roadmap call out the targeting/status labs so every addition appears in the same decision tree and is immediately testable.
- `round_start` events now capture every combatant's HP/max HP, statuses, and abilities so the first log entry documents that each Pokemon truly began at full HP before Burn/Poison ticks or other passive effects fire.
- Rebuilt `dist/AutoPTU.exe` with `pyinstaller --clean AutoPTU.spec` so the fresh binary ships the new status stack; warnings about `tbb12.dll` from numba remain benign.
- New move coverage for the Moon->Night batch (Moonlight through Night Shade): Moonlight/Morning Sun use weather-scaled healing, Mountain Gale/Nature's Madness/Night Shade obey PTU fixed-damage rules, Mud Bomb/Mud Shot/Mud-Slap/Muddy Water apply their stat/drain effects plus Muddy Water's AoE shift, Mud Sport grants Electric resistance until the next Electric hit, Multi-Attack types follow held Memories/Plates/Drives, Natural Gift borrows berry type/DB, Nature Power selects a terrain move, Mystical Fire/Power/logs, Nasty Plot, Needle Arm, and Night Daze replicate the referenced PTU Core 1.05 effects so the CLI logs stay page-accurate (see Chapter 7).
- Implemented the next alphabetic chunk (Recycle through Rock Smash): Recycle reuses the last consumed item or food buff, Reflect/Reflect Type/Refresh add their guard hooks, Relic Song/Rending Spell/Resonance Beam keep their sleep/tick and stage effects, Rest respects Heal Block and forces two rounds of Sleep, Return/Retaliate/Revenge/Reversal scale with loyalty/ally faints/injuries, Revelation Dance mirrors the user's primary type and stacks bonus damage, Riposte readies after a missed melee hit, Rising Voltage creates Electric Terrain once per scene, Roar forces target movement, and the Rock moves log their PTU crit/status effects while the new screen guard processes Reflect/Light Screen/Aurora Veil resistance steps.
- Player turns now support `[C]heck stats` (or typing `check`/`stats`) so the console prints the active Pokemon's HP, stats, combat stages, abilities, statuses, tick value, and movement speeds before you pick an action.
- Confusion, Flinch, Rage, and Trapped now emit skip/outcome rolls, self-hit damage logs, locked-action cues, and trapped warnings; the dedicated `volatile_demo` scenario + Volatile Status Lab launcher option keep the expanded volatile suite replayable whenever the logic changes.
- Added explicit PTU handling for Sacred Fire, Safeguard, Sand Attack/Tomb/Storm, Scald, Scary Face, Screech, Scorching Sands, Searing Shot, and Secret Power (terrain-sensitive results, sandstorm ticks, Safeguard activations, and burn rolls) plus regression tests in `tests/test_battle_state.py`; test suite executed via `python -m pytest tests/test_battle_state.py`.

## [v0.3] ? "Initiative & Turn Structure"

- Documentation scaffolding for the long-term roadmap: created `CHANGELOG.md`, `CONTRIBUTING.md`, and expanded the design docs with PTU Core 1.05 references so junior teammates know where to plug in.
- New initiative queue + turn phase controller (`BattleState.advance_turn/advance_phase`) with CLI logging so every round explains who acts next and why (Speed + d20 + trainer modifiers). Interactive `play` mode now enforces Start - Command - Action - End flow, displays round/turn announcements, and keeps the grid positions in each log entry.
- Added initiative scenario tests to `tests/test_battle_state.py` to prove faster mons, trainer modifiers, and faint skipping behave like PTU Core p. 202-203.
- Multi-trainer turns in the interactive grid session now respect each trainer's `controller_kind`, allowing multiple player-controlled trainers to rotate through initiative instead of being treated as AI.
- Rebuilt the PyInstaller bundle (`pyinstaller --clean AutoPTU.spec`) so `dist/AutoPTU.exe` ships the refreshed initiative queue and controller fixes out of the box.
- Campaign metadata now supports `player_trainers` / `foe_trainers` entries, letting the auto match planner build multiple trainer sides in one plan. The new `multi_trainer_demo` scenario exercises this end-to-end (`auto-ptu play multi_trainer_demo --team-size 2`).
- Targeting now enforces line-of-sight (walls, allies, rough terrain) across the CLI, tests, and AI move selection. Interactive prompts render a highlighted grid showing legal target tiles, while movement previews highlight reachable squares and respect difficult terrain/swim costs, and the target picker lists every ally/enemy with "IN RANGE / LOS blocked / Out of range" status so you can deliberately aim at friendly targets if a move allows it. Push/Pull effects (detected via keywords/text) also slide targets when space allows, logging forced movement results for replays.
- Introduced phase-aware status and ability hooks: Burn/Poison/Badly Poisoned ticks fire automatically at Start/End, Poison Heal converts those ticks into healing, and Shed Skin cures statuses with logged d20 rolls. Scenario tests cover each case and the CLI now narrates the events.
- Synced derived-stat math with the rulebook: `PokemonState.max_hp()` references the Level + (HP A- 3) + 10 formula, damage packets expose STAB/weather bonuses, and regression tests compare our resolver directly against `ptu_engine.calc_damage`.
- Move metadata now separates target distance from AoE footprints. `MoveSpec` carries `target_kind/target_range` plus `area_kind/area_value`, the Fancy PTU CSV importer parses strings such as "6, Ranged Blast 2" and "Burst 1, Friendly", and new scenario tests prove close bursts hit every adjacent mon.
- `UseMoveAction` and the CLI logs now resolve AoE tiles: cones/lines/bursts gather every combatant inside their computed shape, produce one move log per defender with the covered tiles, and the rival AI scores those moves by summing their expected PTU damage (minus friendly fire) before acting.
- Added shared targeting + AI heuristics. All move validations leverage the same range calculator used by the CLI, and the rival AI now scores only legal moves via expected PTU damage (DB + stats + type multiplier) before acting.
- Move pickers and action logs now print Range/AC/DB information plus the exact dice expression and condensed effect text so players can follow PTU Core 1.05 attack steps straight from the console (Chapter 7).
- Interactive play now handles multi-trainer, multi-Pokemon battles on the grid: rosters are grouped by trainer sides, every combatant shows live status/position markers, and both the player and AI can retarget any opponent while the log keeps an auditable position summary.
